# Data Model

## 现状
当前 `dinner-what-to-eat/data/dishes.json` 里的单道菜字段主要有：
- `id`
- `name`
- `photo`
- `category`
- `tags`
- `prepTimeMin`
- `difficulty`
- `caloriesKcal`
- `ingredients`
- `steps`
- `nutritionNote`
- 可选：`sourcePhoto` / `convertedJpg` / `needsConfirm`

这个模型足够支撑“单菜展示”，但不够支撑“晚饭套餐 + 材料检查清单 + 网页端可编辑”。

## 推荐新增两层模型

### 1. dish（单道菜）
保留单菜作为基础原子单位。

建议补强字段：
- `proteinType`: `牛肉` / `鸡肉` / `鱼虾` / `蛋` / `豆制品` / `无`
- `vegTags`: `绿叶菜` / `瓜果类` / `菌菇类` / `根茎类`
- `carbTags`: `米饭` / `面` / `土豆` / `玉米` / `无`
- `editable`: 是否允许网页端直接修改
- `imageStatus`: `original` / `cropped` / `confirmed`

### 2. meal-set（套餐）
新增一层“套餐”模型，表示一顿饭由多道菜组成。

建议字段：
- `id`
- `name`
- `dishIds`
- `proteinDishIds`
- `vegDishIds`
- `carbDishIds`
- `estimatedPrepTimeMin`
- `estimatedCaloriesKcal`
- `shoppingChecklist`
- `notes`

### 3. shoppingChecklist（检查清单）
检查清单应该是从套餐反推出来的可编辑结构。

建议字段：
- `group`: `蛋白质` / `蔬菜` / `主食` / `调味`
- `item`
- `amount`
- `checked`
- `sourceDishId`
- `autoGenerated`

## 页面端应支持的编辑动作
### 单菜编辑
- 改菜名
- 改材料
- 改营养说明
- 改制作时间
- 替换图片
- 标记“已确认/待确认”

### 套餐编辑
- 自动推荐一份套餐
- 手动勾选蛋白质 / 蔬菜 / 主食
- 自动生成检查清单
- 手动修改检查清单

## 推荐演进方式
先不要一次性重构全部数据。

建议顺序：
1. 先让 `dish` 支持更稳定的网页编辑
2. 再新增 `meal-set` 存储结构
3. 最后在晚饭页接入“自动推荐套餐 + 检查清单”
