# Shopping Memory Reminder

## 固定文件
- 使用并维护：`memory/shopping-list.md`
- 若文件不存在：先创建标准模板（分类 + 即将提醒 + 已完成采购）。

## 处理流程
1. 提取购物项：物品名、数量、备注、期望时间（可空）。
2. 自动分类：优先放入最匹配类目（食材日用/厨房带饭/健康药品/电子数码/家居清洁/服饰出行/其他）。
3. 写入清单：用 `- [ ] 物品｜数量｜备注｜提醒时间` 追加或更新。
4. 若提供明确提醒时间：
   - 创建一次性 cron 任务，使用 `openclaw cron add --at <time> --system-event <message> --channel feishu --to user:ou_f83a23d8a1a644713896e41fa5a977f6 --announce`
   - 提醒消息模板：`购物提醒：请购买 <物品列表>。`
   - 同步把提醒写到“即将提醒”区。
5. 用户说“买完了/已买”：
   - 将对应条目标记为 `- [x]`
   - 移动到“已完成采购（归档）”，附完成日期。

## 时间处理规则
- 时间模糊（如“晚上提醒我”）时，先追问明确时间再建提醒。
- 时间明确且可解析时，直接创建提醒并回执任务已创建。
- 若提醒创建失败：保留在“即将提醒”并告知用户失败原因。

## 回复规范
- 回复简短，包含：新增/更新了哪些物品、放到哪个类目、是否已创建提醒。
- 每次完成后，提供“当前待买摘要（3-8条）”。
